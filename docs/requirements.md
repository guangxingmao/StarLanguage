# 星知（StarKnow）Demo 需求文档 v0.1

## 目标与范围
- 目标：在 2-3 周内做出可演示的通识教育 App Demo，验证核心体验与家长/儿童接受度。
- 重点：有内容可看、有题可答、有简单对战和成长可视化。
- 非目标：不做复杂知识图谱和完整社交生态；不做海量内容库。

## 目标人群
- 6-12 岁儿童及其家长（亲子共学）。

## 平台与技术
- Flutter（iOS / Android / Web 可运行演示）。

## MVP 核心功能
1) AI 识图 + 问答（简化版）
   - 入口：上传图片或拍照（Demo 可先用图片选择器）
   - 输出：识别结果 + 1-3 条知识点 + 1 条追问建议

2) 科普内容流（轻量）
   - 卡片式推送（1-2 段短文 + 配图）
   - 支持收藏/点赞（本地或轻量后端）

3) 知识擂台（单人版 + 轻量对战）
   - 单人计时答题（10 题 / 60 秒）
   - 轻量“对战”（演示版可先用 AI 虚拟对手或同机对战）
   - 基础排行榜（本地或伪后端）

4) 成长档案 + 成就系统（基础版）
   - 学习记录：连续天数、答题正确率
   - 成就徽章：首次学习、首次满分、连续 3 天学习

## 后续拓展（暂不做）
- 复杂知识图谱（多层关系、推理）
- 兴趣社群
- 大规模题库与题目自适应推荐
- 家长端精细化管理

## 内容与素材来源（Demo 方案）
- 快速实现：
  - 自建小型内容集（每个主题 10-20 条卡片）
  - 题库与知识点一一对应（每条知识点 2-3 题）
- 风险控制：
  - Demo 先用自建内容
  - 后续考虑授权合作或公开许可内容

## 主题与题库（Demo 建议）
- 主题 1：篮球（规则 + 基础历史 + 简单趣闻）
- 主题 2：历史（中国古代 + 1-2 个朝代概览）

## 数据结构（Demo 级别）
- 使用 JSON/本地资源即可：
  - topics.json（主题列表）
  - facts.json（知识点）
  - questions.json（题库）
  - achievements.json（成就）

## 成功标准（Demo）
- 用户能在 3 分钟内完成：看内容 → 答题 → 看成长档案
- 能展示：识图问答 + 内容卡片 + 擂台 + 成就

## 风险与假设
- 识图效果不足：Demo 允许“识图结果为伪数据/固定示例”
- 内容不足：先小范围内容，保证展示完整链路
- 对战逻辑：先同屏或 AI 虚拟对手

## 开放问题
- 识图与问答用哪种模型 / API
- 内容与图片是否需要统一视觉风格
- 排行榜是否需要联网
